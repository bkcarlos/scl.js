Interface HashDictOptions<K, V>

Options passed to a hash-like dictionary in order to configure its behaviour.

The following example demonstrates a dictionary with a capacity of 1024 and a hashing function that simply returns the integer itself.

const d = HashDict.empty<number, string>({
  capacity: 1024,
  hash: key => key,
  keysEqual: key => key === key,
  valuesEqual: value => value === value
})

Optional keysEqual

keysEqual: undefined | function
  • Defined in dict/hash.ts:42

A predicate for determining when two keys are equal.

Two keys may produce the same hash result, but that does not necessarily mean that they are equal. This function resolves any conflicts.

If omitted, the built-in equality function will be used.

Optional valuesEqual

valuesEqual: undefined | function
  • Defined in dict/hash.ts:52

A predicate determining when two elements are equal.

This function is only called after is has been determined that the keys are equal, so you may safely skip the equality check for the keys.

If omitted, the built-in equality function will be used.
